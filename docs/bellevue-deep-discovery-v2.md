### Paid Parking / Meter Investigation

**No dedicated "paid parking meters" or "parking meter" FeatureServer layer exists publicly** on either AGOL or gis-web. The CBD Parking layer (Layer 95) contains the closest analog: street segments with `PRK_TYPE` text encoding time limits (e.g., "2 Hr Parking 7am - 6pm"). The City's downtown paid parking pilot (Old Bellevue/Downtown) does NOT appear to be backed by a public GIS layer as of this discovery. The `PRZ_TYPE` field in Layer 95 may implicitly represent metered blocks but uses time-limit language rather than meter designation.

---

### Cartegraph / Asset Export Investigation

The `StreetSignMaintenanceRO` service name implies Cartegraph integration (read-only operational maintenance data), but its public layers are reference backgrounds only. The **`All Street Signs` / `Sign Status` layers (136/137)** on Enterprise_Transportation are the operative sign inventory — 19,528 points with `AssetNumber`, `SignTypeDescription`, `TechnicianRemarks`, `OnStreet`, `FromXStreet`, `ToXStreet`, `SideOfRoad`, `Status`. This is the Cartegraph/TIMS asset export rendered publicly. Filter `Status='OPERATING'` AND `SignTypeDescription LIKE '%PARK%'` returns ~4,719 sign points citywide — the richest parking restriction dataset found.

## B. Coverage Projection

Assumptions: Bellevue has ~10,600 street centerline segments (Streets FeatureServer Layer 10 count). Current providers (bellevue-streets + bellevue-rpz + bellevue-curb) cover roughly:
- bellevue-streets: provides geometry base
- bellevue-rpz: ~16 RPZ polygons → ~5-8% of street segments classified as `permit`
- bellevue-curb: 926 BelRed segments → ~8-9% classified

| Scenario | Green (allowed/metered/permit) | Yellow (time_limited) | Orange (no_parking) | Gray (unknown) |
|---|---|---|---|---|
| **Current (baseline)** | ~8% | ~5% | ~2% | **~85%** |
| **+ CBD Parking (Layer 95)** | +1% | +1% | <1% | ~83% |
| **+ RPZ Street Parking (Layer 97)** | — | — | +1% | ~82% |
| **+ Sign Status / All Street Signs (Layer 136/137)** | +5% | +15% | +8% | **~54%** |
| **+ Painted Traffic Curbs (Layer 647)** | +1% | — | +3% | ~50% |
| **+ CurbType expanded (Layer 22 in MoveAutoExist3)** | +2% | +3% | — | ~45% |
| **All above integrated** | ~18% | ~18% | ~12% | **~52%** |

> **Note:** The Sign Status layer (19,528 signs, ~4,719 parking-related) is the single largest coverage unlock. Signs are points, not segments — a spatial join or linear referencing step against street centerlines is required. Each sign covers one curb face; two signs (near/far end of block) are needed to fully classify a segment side.

## C. Recommended Implementation Order

### Priority 1 — High Impact, Moderate Complexity
**`Enterprise_Transportation/MapServer/97` — RPZ Street Parking (polylines)**
- URL: `https://gis-web.bellevuewa.gov/gisext/rest/services/Enterprise/Enterprise_Transportation/MapServer/97/query`
- 118 polylines with full restriction text. Direct street-segment geometry. Drop-in overlay alongside existing bellevue-rpz.
- Restriction code: `permit` (parse hours from `Restriction` field)
- Estimated gain: +1-2% coverage, high confidence classification

**`Enterprise_Transportation/MapServer/95` — CBD Parking (polylines)**
- URL: `https://gis-web.bellevuewa.gov/gisext/rest/services/Enterprise/Enterprise_Transportation/MapServer/95/query`
- 140 polylines covering downtown/Old Bellevue core. `PRK_TYPE` parses to time limits.
- Restriction code: `time_limited` or `no_parking` based on PRK_TYPE text
- Estimated gain: +2-3% coverage in highest-demand area

### Priority 2 — High Impact, Higher Complexity (spatial join needed)
**`Enterprise_Transportation/MapServer/137` — Sign Status (active signs)**
- URL: `https://gis-web.bellevuewa.gov/gisext/rest/services/Enterprise/Enterprise_Transportation/MapServer/137/query`
- 19,528 points, citywide. Filter `Status='OPERATING'`. `SignTypeDescription` is MUTCD code + description.
- Requires spatial join to street segments (match `OnStreet` + `FromXStreet`/`ToXStreet` + `SideOfRoad`)
- Sign codes to map: R7-1 (No Parking), R7-2 (No Parking Any Time), R7-8 (No Stopping), R8-3 (Permit Parking), time-limit signs (R8-series)
- Estimated gain: **+25-35% coverage** citywide — largest single unlock

**`Enterprise_Transportation/MapServer/647` — Painted Traffic Curbs**
- URL: `https://gis-web.bellevuewa.gov/gisext/rest/services/Enterprise/Enterprise_Transportation/MapServer/647/query`
- 915 polylines. YELLOW = No Parking / Loading / Bus, WHITE = Passenger Loading.
- `Color` + `CurbType` → restriction code
- Estimated gain: +3-5%

### Priority 3 — Supplemental / Niche
**`Enterprise_Transportation/MapServer/108` — Bus Layover Zones**
- 7 polygons, geometry-only. Use as spatial overlay to tag `bus_zone`.

**`CurbType_MoveAutoExist3/FeatureServer/22` — Expanded curb typology**
- 2,004 features vs 926 in current bellevue-curb. Same schema. Replace or supplement existing layer.
- Gain: ~2x BelRed area coverage

**`TIMS_Reference/MapServer/4` — Street Blocks (4,429 features)**
- Better linear referencing backbone than Layer 11 (segments). Use for sign-to-segment matching.

### Priority 4 — Validation / Enrichment Only
- `Citations/FeatureServer/0` — 80,603 citation points. Use as confidence heatmap to validate restriction accuracy (high citation density → likely restricted zone).
- `TIMS_Reference/MapServer/11` — Street Segments (5,683). Cross-reference with Streets FeatureServer for segment ID matching.
